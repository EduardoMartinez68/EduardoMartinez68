🆘 <plus-help> Custom Element

👉 What is <plus-help> and What Is It Used For?

The <plus-help> custom HTML element is used to display interactive step-by-step tutorials inside your ERP system. It's great for guiding users on how to use specific features or complete actions, such as charging with a receipt or navigating a module.

✅ Ideal for: onboarding new users, providing contextual help, and enhancing the usability of complex ERP features.

⚙️ How to Use It in HTML


<plus-help 
  title="How to charge with receipt?" 
  data='help-login' 
  t="label-search-home" 
></plus-help>
  • title: The default text for the help button.
  • data: The unique key used to link the help steps.
  • t: (optional) A translation key if the title needs translation.
---

🔨 How to Add Help Steps with JavaScript

You can add help steps dynamically using either of the following constructors:

🧱 Method 1: StepHelp


const help = new StepHelp('help-login');
help.title = translate_text('label-search-home');
help.message = 'Click on "Add products" and select the items for the receipt.';
help.image = '';
help.video = 'https://www.youtube.com';

🧱 Method 2: CreateHelp (Recommended)


const help = new CreateHelp('help-login');

help.add_step({
  title: 'Step 1: Search Client',
  message: "Use the top search bar to find and access the client's profile."
});

help.add_step({
  title: translate_text('label-search-home'),
  message: "Click on 'Add products' and select the items for the receipt."
});

help.title = 'Help Tutorial: Charging with Receipt';
help.message = 'Follow the steps to complete a sale with a receipt.';
help.image = '';
help.video = 'https://www.youtube.com';

help.save_help_data();
---

🧠 How It Works Internally

  • Each <plus-help> instance creates a button with the title text.
  • When clicked, it shows a popup modal with help steps.
  • It reads from PLUS_HELP_DATA['ayuda-recibo'] to render the tutorial steps.
  • Each step can include:
    • ✅ Title
    • ✅ Text description
    • ✅ Optional image
    • ✅ Optional embedded YouTube video

🗂️ How Steps Are Rendered in the DOM

Each step is shown in a modal popup with forward/back navigation and a progress bar. Here’s how it looks inside the modal:


<div class="plus-help-step">
  <h3>Step 1: Search Client</h3>
  <p>Use the top search bar to find and access the client's profile.</p>
  <img src="/my_app/static/img/search_client.png" class="plus-help-img" alt="Step Image">
  <iframe src="https://www.youtube.com/embed/dQw4w9WgXcQ" class="plus-help-video"></iframe>
</div>
---

🔒 Authentication Requirements

By default, the system ensures users are logged in to access help content. This is controlled by decorators in the auto-generated views.py.

To disable this login requirement for a specific app, modify the config.yaml of the app.

---

🎯 Final Tips

  • Use short, clear messages for each step.
  • Embed helpful images or videos to improve understanding.
  • Reuse translation keys for consistency across the app.
  • Remember to call save_help_data() to store the help steps in PLUS_HELP_DATA.
