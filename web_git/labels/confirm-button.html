To create a confirmation button in the ERP, use <confirm-button>

The custom component <confirm-button> creates a button that asks for user confirmation before running an action. It supports translations via dictionary keys.

ðŸ”§ Basic Structure

Attribute Required? Description
title No Popup title. Can be a static string or a translation key.
message No Message shown in the confirmation dialog. Can be static or a translation key.
onclick Yes Function to call if the user confirms the action. Must be defined globally in window.
textContent Yes Text that appears inside the button (e.g., Delete, Save, etc).

ðŸ“Œ Examples

You can use static strings or translated keys in title and message:


  --- With static title and message ---
  <confirm-button 
    title="Delete Product" 
    message="Are you sure you want to delete this product?" 
    onclick="deleteProduct()">
    Delete
  </confirm-button>

  --- With translation keys ---
  <confirm-button 
    title="delete.title" 
    message="delete.confirmation.message" 
    onclick="deleteItem()">
    {{ 'button.delete' }}
  </confirm-button>


ðŸ“˜ Notes

  • You can define your translations in translate.json and LANG object to make it work with multilingual support.
  • The onclick must be a global function available on the window (e.g., window.deleteProduct = function () {...}).
  • The confirm popup uses a customizable overlay and styles. Make sure your CSS includes classes like .confirm-popup, .popup-buttons, .popup-accept, .popup-cancel, etc.
